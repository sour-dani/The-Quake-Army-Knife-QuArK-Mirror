This snapshot contains only bugfixes to the snapshot 0728.<br>
We are on the way to QuArK 6.1 , and this is one step towards it.<br>
<br>
Changes:
<ul>
<li>.map extension save fix
<li>Added exception messages for debugging purposes
   (in order to catch the stream read errors)
<li>fixed kingpin texture flip
<li>tiglaris bezier join fix
<li>texture coordinate pyton crash fix
</ul>
Q3A:
<ul>
<li>added missing Quake 3 entites
<li>fixed incorrect comments
<li>fixed incorrect specifics,
<li>added missing specifics,
<li>added model2 menu for func_* entities,
<li>more logical placement of entities,
<li>removed incorrect or broken entities.
</ul>
SOF:
<ul>
<li>fixed default texture for SoF
<li>added sky "ny" specific to worldspawn
</ul>
<br>
INSTALLATION: <br>
<br>
<ol>
<li>Make sure, that you have at least the mini python package
   installed. Get it at <a href="download.php">download page</a>
   The full python install does it also (<a href="http://www.python.org">http://www.python.org</a>)
<li>Unpack the ace archive with e.g. xaceplus from <a href="http://www.winace.com">www.winace.com</a>
   A directory will be created.
<li>Start QuArK by clicking quark.exe
<li>Don't forget to configure the Game directories and tools.
   If you don't have the map compiler tools for the game you are
   making maps for, please refer to
     <a href="download.php">download page</a>
<li>When making maps for Q3, check if you have the mapmedia.pk3
   file (from QERadiant) in your baseq3 directory. Otherwise
   some shader textures and models are missing.
<li>Please, when reporting bugs, state for which game the report
   is. Often it is game specific.
</ol>
Unpacking the snapshot over an older release may work at the first
glance, but may lead to subtile errors that are hard to find.<br>
I do not recommend it.<br>
