<headline>Problem using <i>Game Context</i> and Half-Life Mods...</headline>
It has been brought to my attention, that using QuArK's 'Game Context' in Half-Life mode, causes problems with Half-Life modifications, when QuArK is used to start Half-Life. <img src="pics/SmileySad.gif" align=absmiddle width=15 height=15>
This is due to a <i>feature</i> that allow QuArK to create/copy files, that is needed by Half-Life to run. Basically its these two files that causes the problem:<br>
&nbsp;&nbsp;&nbsp;HALF-LIFE/&lt;moddir&gt;/LIBLIST.GAM<br>
&nbsp;&nbsp;&nbsp;HALF-LIFE/&lt;moddir&gt;/CL_DLLS/CLIENT.DLL<br>
They get overwritten when QuArK runs Half-Life, thereby destroying the modification's custom files.<br>
There is a quick fix for this, but that will render the HALF-LIFE/tmpQuArK/ directory useless, unless you remember <u>not</u> to delete the HALF-LIFE/tmpQuArK/LIBLIST.GAM file!<br>
The fix is to manually go into the following three QuArK text-files, and delete the lines shown.
But take care! I suggest you take a <u>backup of these files</u>, just in case, so you can restore them if anything goes wrong.<br>
If you have <u>absolutely no clue</u> of what I'm talking about, then don't do anything - other than wait for the QuArK 6.00 beta-2 release.
<pre>
<i><u>DataHL.QRK</i></u>
  CreateFiles: =
  {
    liblist.gam = {
      Data =
        "game "$22"tmpQuArK"$220D0A
        "gamedll "$22"..\valve\dlls\hl.dll"$220D0A
        ".\valve\dlls\hl.dll"$0D0A
    }
  }

<i><u>Defaults.QRK</i></u>
  ExtractFiles = "cl_dlls/*.*"

<i><u>Setup.QRK</i></u>
  ExtractFiles = "cl_dlls/*.*"
</pre>
